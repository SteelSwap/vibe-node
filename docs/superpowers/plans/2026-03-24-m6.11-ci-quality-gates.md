### Task 4: Add CI lint/format workflow

**Files:**
- Create: `.github/workflows/lint.yml`

- [ ] **Step 1: Create lint workflow**

```yaml
name: Lint & Format

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install uv
        uses: astral-sh/setup-uv@v6
        with:
          enable-cache: true

      - name: Set up Python
        run: uv python install 3.14

      - name: Install dependencies
        run: uv sync --dev

      - name: Ruff check
        run: uv run ruff check src/ packages/ tests/

      - name: Ruff format check
        run: uv run ruff format --check src/ packages/ tests/

      - name: Black check
        run: uv run black --check src/ packages/ tests/
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/lint.yml
git commit -m "ci(m6.11): add lint + format CI workflow"
```

### Task 7: Write code style guide

**Files:**
- Create: `docs/reference/code-style.md`

- [ ] **Step 1: Write code style guide**

Content must cover:
- **Import ordering** — stdlib, third-party, local (`from vibe.*`). Enforced by ruff isort.
- **Type annotations** — required on all public API functions and class methods
- **Logging conventions** — use `logging.getLogger(__name__)`, structured `extra={}` fields for events
- **Error handling** — custom exceptions, no bare `except:`, catch specific exception types
- **Dataclass usage** — `@dataclass(frozen=True, slots=True)` by default
- **CBOR usage** — always `import cbor2pure as cbor2`, never use `cbor2` directly (buggy C bindings)
- **Naming** — snake_case for functions/variables, PascalCase for classes, UPPER_CASE for constants
- **Haskell references** — include `Haskell ref:` comments when implementing from Haskell source

- [ ] **Step 2: Commit**

```bash
git add docs/reference/code-style.md
git commit -m "docs(m6.11): add code style guide"
```
