### Task 4: Write Phase 2-6 roadmap with testable deliverables

**Files:**
- Modify: `docs/reference/architecture-blueprint.md`

- [ ] **Step 1: Define each phase with concrete deliverables**

For each phase:

**Phase 2 — Serialization & Networking**
- Deliverable: Decode real mainnet blocks, sync headers via chain-sync
- Subsystems: Serialization (pycardano CBOR), Networking (asyncio multiplexer), Chain-sync client
- Test gate: decode 10,000 mainnet blocks, hash matches Haskell output
- Estimated scope from rules: X rules in serialization, Y in networking

**Phase 3 — Chain Sync & Storage**
- Deliverable: Full block sync from Mithril snapshot, validate pre-Plutus eras
- Subsystems: Block-fetch, Storage (Arrow+Dict), Ledger (Byron-Mary)
- Test gate: sync to tip on preprod, agree on block validity for Byron-Mary

**Phase 4 — Ledger & Consensus**
- Deliverable: Full ledger validation (all eras), tip selection agreement
- Subsystems: Consensus (Praos), Ledger (Alonzo-Conway), Plutus (uplc)
- Test gate: agree with Haskell on tip within 2160 slots for 24h

**Phase 5 — Block Production & N2C**
- Deliverable: Produce valid blocks, serve local queries
- Subsystems: Block production, Mempool, N2C miniprotocols
- Test gate: produce blocks accepted by Haskell nodes on preview/preprod

**Phase 6 — Hardening**
- Deliverable: Production readiness
- Test gate: 10-day soak test, memory ≤ Haskell, power-loss recovery
